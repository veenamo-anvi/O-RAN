## High-Level Idea

Build a multi-agent Agentic AI system that accepts deployment parameters for a specific
geographic area — Malleswaram, North Bangalore — and automatically plans, configures,
deploys, or modifies an Open RAN (O-RAN) compliant 4G/5G network. The system is
context-aware of the live deployment and continuously optimizes for load balance and
operator efficiency. The live deployment is simulated by 3 DU containers and 1 CU container
covering a population of 40,000 residents + 15% commuter overhead (46,000 effective), with
18,400 peak active UEs at 40% operator market share. All UE telemetry is synthetic,
generated by the DU simulators using realistic hardware specifications and diurnal load
curves.

- **Orchestrator** — Understands operator commands via an LLM and routes them to the right
  tools; streams the response back to the operator.
- **Controller** — The single source of truth for the network topology; handles live
  cell/DU moves and topology CRUD.
- **Planning Agent** — Designs new network layouts for cell placement and DU/CU grouping
  using placement algorithms, PCI planning, and slice allocation. UE counts and hardware
  limits are calibrated to real market equipment specs.
- **KPI Agent** — Continuously monitors cell health using a bidirectional LSTM model,
  detects anomalies (overload, poor signal, power waste), and takes autonomous corrective
  actions.
- **Map Server** — A live map container showing the current cell layout with per-cell KPI
  overlays, colour-coded by vendor and generation.

The RAN is a 4G/5G NSA mix — both 4G LTE and 5G NR connect to the same NSA core (shared
AMF/SMF/UPF). 5G NR deployment uses split CU/DU/RU architecture throughout. The KPI agent,
controller, or planning engine can reorganise cells at any time in response to changing load
or operator instruction.

Project phase: Development.
